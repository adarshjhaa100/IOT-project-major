Description:
Abstract:
Brackish groundwater (BGW), defined for this assessment as having a dissolved-solids concentration between 1,000 and 10,000 milligrams per liter is an unconventional source of water that may offer a partial solution to current (2016) and future water challenges. In support of the National Water Census, the U.S. Geological Survey has completed a BGW assessment to gain a better understanding of the occurrence and character of BGW resources of the United States as an alternative source of water. Analyses completed as part of this assessment relied on previously collected data from multiple sources, and no new data were collected. One of the most important contributions of this assessment was the creation of a database containing chemical data and aquifer information for the known quantities of BGW in the United States. Data were compiled from single publications to large datasets and from local studies to national assessments, and includes chemical data on the concentrations of dissolved solids, major ions, trace elements, nutrients, radionuclides, and physical properties of the resource (pH, temperature, specific conductance). This dataset represents major-ions data from a compilation of water-quality samples from 16 sources for about 124,000 groundwater wells across the continental U.S., Alaska, Hawaii, Puerto Rico, the U.S. Virgin Islands, Guam, and American Samoa. The data are published here as an ESRI geodatabase with a point feature class, and associated attribute table, and also as non-proprietary comma-separated value table. Major-ions data include information for assessing the geochemical-water type, saturation indices, and potential for mineral scaling. It was not possible to compile all data available for the Nation, and data selected for this investigation were mostly limited to larger datasets that were available in a digital format. As a result, some data on a more local-scale may not be included.
Purpose:
This dataset was compiled for the purpose of assessing the geochemistry of brackish groundwater across the United States and its territories. Please note that there is a related dataset (https://doi.org/10.5066/F72F7KK1) containing dissolved-solids data that was used for assessing the distribution of brackish groundwater across the United States and its territories.
Supplemental Information:
Data-qualifier codes and brief method descriptions are included with this dataset when possible to provide supporting information for each sample and/or value. Description fields (_desc) were used to convey information about each parameter, such as analysis methods (field is left blank (NULL) if no information was provided by the source). The _desc field is formatted as "[NWIS parameter code]_[constituent] "water" [filtration/method] [units]". Where possible, the values were matched with National Water Information System (NWIS) parameters if enough information about the analysis method was known and the NWIS parameter matched is in the description field. If nothing was known about the methodology, then the non-match with the NWIS parameter is noted. Data-qualifier codes were obtained from original sources if available. Data qualifiers were added if geochemical data values contained inconsistencies (as determined for this assessment) but were not identified by the original source. Definitions of data-qualifier codes (used to populate the “flag” fields) are below (Note: multiple codes may occur in combination within the flag field; this field is left blank if the values are not flagged):
<  Actual value is known to be less than the value shown
>  Actual value is known to be greater than the value shown
M  Presence of material verified but not quantified
U  Analyzed for, not detected
V  Contamination analyte was detected in both the environmental sample and the associated blanks.
T  Concentration of parameter is greater than TDS_mgL field (associated value removed)
E  Value is estimated
A  Value is an average
BB  Unknown meaning (from original source)
N  Presumptive evidence of presence of material
cull  Original value from data source marked as cull (associated value removed)
Q  Unknown meaning (from original data source)
H  Unknown meaning (from original data source)
ppb  Original value in parts per billion and no specific gravity supplied to convert to micrograms per liter
ppm  Original value in parts per million and no specific gravity supplied to convert to milligrams per liter
-3333  NA, Not Analyzed, or Unknown
-4444  Transcription error or otherwise nonsensical entry (associated value removed)

Values are flagged with an "x" if they meet the following criteria:
pH: Field and lab values from the same sample differ by more than one pH unit, or the pH value is less than 4 or greater than 10.5.
Dissolved oxygen: Value is less than 0 or greater than 20 milligrams per liter (Dupré and others, 2013, Appendix M).
Dissolved solids: Value was less than 7 mg/L or the original value was flagged.
Temperature: Value is less than 1 or rather than 100 degrees Celsius.
Specific conductance:  Lab and Field values differ by plus or minus 10 percent (Dupré and others, 2013, Appendix M) or the value was less than 10 microsiemens per centimeter or the original value was flagged.
All other fields: Data value was set to zero but no data-qualifier code was provided by the source (used for fields "Alk_mgL", "Ca_mgL", Cl_mgL", "Dens_gmL", "HCO3_mgL", "Mg_mgL", "Na_mgL", "ph", "SC_uScm", "SO4_mgL", and "TDS_mgL").

Data Quality Information:

Attribute Accuracy:
Attribute Accuracy Report:
Most sources of groundwater-chemistry data did not provide documentation about sampling, preservation, analytical, and quality-assurance techniques. Several checking routines were used to identify systematic errors in the data and remove those values that were deemed incomplete or unreliable (see the “Data and Methods Used for Analyses” section of USGS professional paper 1833 (https://doi.org/10.3133/pp1833). However, errors are still likely to exist within the datasets. Data from some sources were originally provided as provisional with the expectation that the data may contain inaccurate information. In addition, several errors in the data were brought to the attention of the authors after the analyses were completed. Locations for wells were checked for accuracy and a flag was used (“location_flag”) to represent the reliability of the location information for both the source dataset (was datum known, were locations from GPS, and so forth) and individual well locations (is the well location plotting within the state that is indicated for the sample). See “Horizontal Positional Accuracy Report” section of this metadata for details about the location flag. Also see USGS Professional Paper 1833 (https://doi.org/10.3133/pp1833) section "Geochemical Data". None of the data have been independently verified as part of this assessment’s methods. Despite these limitations, it is expected that data errors represent a small part of the thousands of records, and the generalized results provided within this report are reasonably accurate within the context of a national-scale assessment.
Logical Consistency Report:
All wells selected for analyses were required to include geographic-location coordinates or information that could be used to derive coordinates and either a well-depth measurement or contributing aquifer information. Samples from qualifying wells were selected if they included a measured value for pH, temperature, alkalinity, calcium, chloride, magnesium, potassium, sodium, and sulfate. The most recent sample meeting these requirements was selected for the dataset. Although the same well may exist in the companion dissolved-solids dataset, a different sample was often selected for this dataset based on different criteria.
Many of the data sources for this dataset are compilations, and samples are known to be duplicated within and among the sources. If a well appeared to be a duplicate based on location (within 1,000 ft of another well) and depth (within 5 percent difference), then the sample date, dissolved solids, specific conductance, calcium, chloride, potassium, sodium, and magnesium values were compared. If the sample results matched for those values, one of the samples was removed from the data used for analyses.
Please see the "Process Steps" section of this metadata for a more detailed discussion.
Completeness Report:
These data were obtained from a wide variety of sources; however, data representing deeper, more saline groundwater were not as readily available and the compiled data are biased toward shallow and fresh water resources (well depth (ft) – 62, 145, 323; 25th, 50th, and 75th percentiles respectively, dissolved-solids concentration (mg/L) – 233, 406, 832; 25th, 50th, and 75th percentiles respectively). It was not possible to compile all data available for the Nation, and data selected for this investigation were mostly limited to larger datasets that were available in a digital format. As a result, some data on a more local-scale of brackish groundwater resources may not be represented. For some groundwater samples, results from multiple methods were reported for a chemical constituent, but a single value was selected for analyses; for example, a sample may include a calcium-concentration result from a filtered and also an unfiltered groundwater sample. In these cases, a prioritization scheme was used to select the best result value available for a sample. In general, filtered samples were selected over unfiltered samples, and common laboratory methods were selected over uncommon laboratory or field methods. For pH and SC, field methods were selected over laboratory methods. Wells were removed from the datasets used for analyses if the selected value to represent maximum depth was less than zero or were not in agreement with other types of depth values for the well; for example, a depth value was not deemed reliable if the total well depth was shallower than depth to the bottom of a well screen. Values of individual constituent measurements were removed if the assigned data-quality codes indicated the value was suspect (data-quality codes "x", "T", -4444, or "cull"), however, the data-quality code remains. The concentration measurement of a major ion, trace element, or nutrient was removed if it was higher than the dissolved-solids concentration of the same sample. Samples that were questionable because they had a dissolved-solids concentration that either did not agree with other dissolved-solids concentrations from different analytical methods for the same sample (that is, if the ratio between the values was >1.12 or <0.90) or was less than the dissolved-solids concentration associated with rainwater, were removed from the dissolved-solids dataset. Also see USGS Professional Paper 1833 sections "Data Sources," "Geochemical Data Selection," and "Data Gaps and Study Limitations".

Horizontal Positional Accuracy:
Horizontal Positional Accuracy Report:
Attempts were made to assess the location information for all of the wells where samples were collected. A data-quality code was created called "location_flag" that is a compilation of 2 parts: [Location overview].[Location quality flag]. Location overview conveys what is known about the original accuracy information from the data source (samples with a Location quality flag of 3 and 4 were removed from the dataset): 1 –Datum provided, and location accuracy is reported to be known within 10 seconds. (Only USGS sites meet this criterion because other sources don’t generally provide any indication of the location accuracy) 2 – Datum provided but no location accuracy information 3 – Datum unknown, and location accuracy unknown, but some information is provided about source of latitude/longitude (therefore can make some educated assumptions about the location accuracy) 4 - Datum unknown, or location accuracy unknown, or location accuracy known to be greater than 10 seconds, or location derived from data other than latitude/longitude, such as legal description (such as cadastral or PLSS data). The location quality flag conveys information on where location plots on a map given the location stated in the database: 0 – No issues; location is plotting in the state specified in the database. 1 – Good; location of well plots right along state border or shoreline within the probable accuracy of the state/shoreline dataset used for identifying accuracy of plotting location. Also used for produced water wells that plot offshore but are assumed to be oil/gas wells or test holes. 2 – Fair; well plots slightly further away from state border or shoreline (beyond possible accuracy issues) but still close enough that the state MAY have sampled across the border for some reason or the latitude/longitude is slightly off. May still probably use for analysis but be aware if anything looks different about the data. Special cases may be the Produced Waters version 2.1 dataset, where the latitude/longitude may be for the center of an oil field, in which case, the wells may fall over the border in a neighboring state than the one identified. 3 – Questionable; well plots more further into neighboring state than you would expect for sampling from the bordering state or the well plots well offshore or plots in Canada. Use at your discretion. 4 – Bad; the well plots significantly off the coast or elsewhere in the world. Assumed to be a completely wrong latitude/longitude, transcription error, transposition error, and so forth. Probably should not use.

Process Step:
Process Description:
Digital tabular data from the sources listed above were received in various formats. Tables (generally in text file or Microsoft Excel file formats) were imported into Microsoft Access. A series of queries was used within Microsoft Access to put each data table into common units and format for inclusion into a master database. For many fields, a description field was added to hold information about the original units of measurement, method descriptions, and/or comments that were in the original datasets from the original sources. Therefore, units in the description field may no longer match the data value. Measurements provided in parts per million (ppm) or parts per billion (ppb) were converted to milligrams per liter or micrograms per liter, respectively, when a specific gravity was provided by multiplying the original value by the specific gravity. If a specific gravity was not provided the original value was kept and coded with either "ppm" or "ppb". For parameters expressed as, for example,  “as nitrogen,” values were converted, if needed, using a conversion factor so as to all be in units of “as nitrogen”.

Process Step:
Process Description:
Data from all sources were imported to a Structured Query Language (SQL) server database. Their data format was restructured to match the USGS National Water Information System (NWIS) data model because it represented the largest number of samples in the dataset. This model includes a well description table, a data sample table, and data measurement table. NWIS parameter descriptions were used to describe data measurements when feasible and new parameter descriptions were created when there was not sufficient information to assign them to an existing parameter.

Process Step:
Process Description:
For some groundwater samples, results from multiple methods were reported for a chemical constituent, but a single value was selected for analyses; for example, a sample may include a calcium-concentration result from a filtered and also an unfiltered groundwater sample. In these cases, a prioritization scheme was used to select the best result value available for a sample. In general, filtered samples were selected over unfiltered samples, and common laboratory methods were selected over uncommon laboratory or field methods. For pH and SC, field methods were selected over laboratory methods. When possible, the method description is including in the “_desc” field associated with each chemical constituent.

Process Date: 11/01/2014
Process Step:
Process Description:
Aquifer and brackish groundwater regions were assigned to wells using a combination of automated spatial and manual visual inspection methods. Geospatial datasets of the horizontal and vertical extents of principal aquifers were obtained primarily from previously published USGS reports that were part of the RASA studies (Sun, 1986), Groundwater Availability Studies (http://water.usgs.gov/ogw/gwrp/activities/gw-avail.html), or the Ground Water Atlas of the United States (Miller, 2000) (http://pubs.usgs.gov/ha/ha730/gwa.html). The source citations for all of the aquifers used for the automated method can be found in the table "AquiferExtentSources.xlsx" served with this dataset. The USGS aquifer extent datasets were used to automatically assign an aquifer to wells by location and depth using a GIS. This was completed by comparing well-depth and geologic-formation information with the horizontal and vertical boundaries of the principal aquifers. For wells where this was not possible (no geospatial aquifer extent was available or multiple aquifers were possible), the wells were manually inspected within the GIS to assign aquifers to wells contained by more than one aquifer or those with origination information conflicting with the spatial assignment. Aquifer assignment from the original source was used when available.
Process Date: 12/01/2014
Process Step:
Process Description:
The preliminary comma-delimited text file was processed using R® software; an environment for statistical computing and graphics. Temperature, pH, and dissolved oxygen values were identified that were extreme or conflicted with other values of the same parameter. Dissolved solids and specific conductance (SC) values were identified that were extreme or conflicted with each other. Dissolved solids values were calculated from specific conductance values where dissolved-solids concentration, in milligrams per liter was equal to −55+0.689*SC for specific conductance values less than 50,000 microsiemens per centimeter at 25 degrees Celsius (approximately seawater) and dissolved-solids concentration, in milligrams per liter was equal to 27,720−0.0869*SC+6.204*10∧-6*SC∧2 where SC was greater than 50,000 microsiemens per centimeter at 25 degrees Celsius. Values were censored where SC was less than 150 microsiemens per centimeter.
Process Date: 01/01/2015
Process Step:
Process Description:
A cluster analysis that implemented the k-means algorithm (MacQueen, 1967; SAS Institute, Inc., 2008) was used to find groups of samples where measures of basic geochemistry were similar within each group, but also were very different from that of samples in other groups, thereby creating geochemically distinct groups. The geochemical group was appended to the major-ions dataset in the attribute field named ”geoch_group”.
Process Date: 02/01/2015
Process Step:
Process Description:
The the pH-REdox-Equilibrium computer program (PHREEQC) (Parkhurst and Appelo, 1999) was used to (1) calculate constituent speciation and mineral saturation indexes (SIs)(Parkhurst and Appelo, 2013; Charlton and Parkhurst, 2011) and (2) simulate processes including dilution, evaporation, and a hypothetical treatment process resembling reverse osmosis (RO) . To simulate evaporation, pure water was removed from solution to produce concentration factors of 1 (0 percent of the water removed), 2 (50 percent of the water remove), 4 (75 percent of the water removed), 8 (87.5 percent of the water removed), and 16 (about 94 percent of water removed)(represented by the "_1x", "_2x", "_4x", "_8x", and "_16x" appended to the field names representing modeled amounts of precipitate). Simulations were completed under open (pCO2 = 10^-3.4) and closed (no pCO2) atmospheric conditions with respect to the partial pressure of carbon dioxide (pCO2). The first simulation step (step “0”) was completed under approximated “groundwater conditions” of 15 degrees Celsius and a pressure of 1 atmosphere. Subsequent steps to approximate RO were completed at 25 degrees Celsius and pressures of 20 atmospheres (also see U.S. Geological Survey Professional Paper 1833, section "Analytical Methods/Geochemical Characterization").

Process Description:
The arcpy (Python version 2.7) library was used in a python script to load csv files into an ArcGIS geodatabase. A schema.ini file was created to describe the columns and their associated data types and was placed in the same directory as the csv files. The files were loaded into the python session as tables, then converted to a feature layer with a spatial reference of NAD83. The resulting layers were projected to USA Contiguous Albers Equal Area Conic projection and saved to the geodatabase. The code used is as follows:
import arcpy

arcpy.env.workspace = "F:/Alta/Brackish/BrackishWater_QC.gdb"

for cfq in fclistq:
arcpy.Delete_management(cfq)

arcpy.env.workspace = "F:/Alta/Brackish/BrackishWater.gdb"
fclist = arcpy.ListFeatureClasses() + arcpy.ListTables()

for cf in fclist:
arcpy.Delete_management(cf)

spref =  r"Favorites\Nad 1983"
sprefNew =  r"Coordinate Systems\USGS_Favorites\USA Contiguous Albers Equal Area Conic USGS.prj"

arcpy.TableToTable_conversion("F:/Alta/Brackish/BrackishFiles/BrackishFiles_2015_12_31b/Diss_Solids_2015_12_31_final.csv", "F:/Alta/Brackish/BrackishWater.gdb", "TDS_filt_p")
arcpy.TableToTable_conversion("F:/Alta/Brackish/BrackishFiles/BrackishFiles_2015_12_31b/Major_Ions_2015_12_31_final.csv", "F:/Alta/Brackish/BrackishWater.gdb", "MajorIons_filt_Phreeqc_p")

arcpy.MakeXYEventLayer_management("MajorIons_filt_Phreeqc_p", "dec_long_va", "dec_lat_va", "XY_MajorIons_filt_Phreeqc_p", spref)
arcpy.CopyFeatures_management ("XY_MajorIons_filt_Phreeqc_p", "XY_MajorIons_filt_Phreeqc_p_save")

arcpy.Project_management("XY_TDS_filt_p_save", "Diss_Solids", sprefNew)

arcpy.Delete_management("XY_MajorIons_filt_Phreeqc_p_save")
arcpy.Delete_management("MajorIons_filt_phreeqc_p")
arcpy.Delete_management("XY_MajorIons_filt_Phreeqc_p")
